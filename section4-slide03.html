SPEC BASICS

A SPEC IS A
CONTRACT

A spec (specification or PRD) is a written description of what you want to build: goals, constraints, acceptance criteria, and edge cases — BEFORE you touch a prompt.

BAD PROMPT

"Make a login form that works well"

SPEC

Build a login form with:
— email + password fields
— validation on submit
— error messages below each field
— disabled submit while loading
— redirects to /dashboard on success

The spec is the source of truth. The AI is the builder.
