API KEYS WITH ASTRO:ENV/SERVER

API keys must be protected on the server. Astro provides astro:env/server to manage environment variables securely, preventing them from being exposed on the client. 

// astro.config.mjs
import { defineConfig, envField } from 'astro/config';

export default defineConfig({
  env: {
    schema: {
      GEMINI_API_KEY: envField.string({
        context: 'server',
        access: 'secret'
      })
    }
  }
});

Nunca expongas API keys en el código del cliente
