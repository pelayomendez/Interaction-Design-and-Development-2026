SDD RULES
TO LIVE BY

These habits separate developers who use AI effectively from those who fight it.

Always write the spec before the first prompt
Version your specs alongside your code (Git)
Use Claude to REVIEW your spec, not just build from it
Keep specs small and focused — one feature per file
When output drifts, fix the spec, not just the prompt
Share specs with teammates, not just prompts

A spec is a communication tool. It works with humans too.
