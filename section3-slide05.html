VIRTUAL ENVIRONMENTS

Virtual environments (venv) isolate dependencies per project, avoiding conflicts between package versions. 

# Create virtual environment
python -m venv venv

# Activar (macOS/Linux)
source venv/bin/activate

# Activar (Windows)
venv\Scripts\activate

# Instalar dependencias
pip install -r requirements.txt

# Congelar dependencias actuales
pip freeze > requirements.txt

# Desactivar entorno
deactivate

Best practice: Always use venv for Python projects
