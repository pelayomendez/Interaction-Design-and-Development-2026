NOSQL

WHEN TABLES
DON'T FIT

NoSQL databases trade the strict structure of SQL for flexibility. Instead of rows and columns, they store documents, key-value pairs, or graphs.

SQL (RELATIONAL)

users table:
┌────┬──────┬───────────────┐
│ id │ name │ email         │
├────┼──────┼───────────────┤
│  1 │ Ana  │ ana@test.com  │
│  2 │ Bob  │ bob@test.com  │
└────┴──────┴───────────────┘
Strict schema. Every row same shape.

NOSQL (DOCUMENT)

// MongoDB / Firestore document
{
  "id": "user_001",
  "name": "Ana",
  "email": "ana@test.com",
  "preferences": {
    "theme": "dark",
    "notifications": true
  },
  "tags": ["admin", "beta"]
}
// No fixed schema. Each doc differs.
