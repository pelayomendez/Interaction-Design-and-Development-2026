Intro

Why TypeScript?

JavaScript with a safety net

TypeScript is a superset of JavaScript that adds static types.

Without types:

  • Bugs hide until runtime
  • No autocomplete for object shapes
  • Refactoring is risky
  • APIs are documented in comments (maybe)

With TypeScript:

  • Errors caught as you type
  • Autocomplete knows every property
  • Rename a field, find all usages instantly
  • Types are the documentation
before.js 
// JavaScript — no safety
function getArea(shape) {
  return shape.width * shape.height;
}

// Oops — typo, no error until runtime
getArea({ widht: 10, height: 5 });
// NaN (silent bug!)
after.ts 
// TypeScript — caught immediately
interface Rectangle {
  width: number;
  height: number;
}

function getArea(shape: Rectangle): number {
  return shape.width * shape.height;
}

// Error: 'widht' does not exist
getArea({ widht: 10, height: 5 });
//        ^^^^^ caught at compile time!
Types 1

Basic Types

TypeScript adds type annotations after variable names with a colon.

string

Text values

number

All numeric values

boolean

true or false

any / unknown

Escape hatches (use sparingly)

Key concepts:

  • Type inference — TS guesses types when obvious
  • Union types — string | number
  • Literal types — "success" | "error"
  • Arrays — string[] or Array<string>
types.ts 
// Primitive types
let name: string = "Alice";
let age: number = 28;
let active: boolean = true;

// Type inference — annotation optional
const score = 100; // inferred as number

// Arrays
const tags: string[] = ["js", "ts"];
const ids: Array<number> = [1, 2, 3];

// Union types
let id: string | number = "abc";
id = 123; // also valid

// Literal types
type Status = "idle" | "loading" | "error";
let state: Status = "idle";

// Tuples
const pair: [string, number] = ["Alice", 28];
Types 2

Interfaces & Type Aliases

Define the shape of your data with interfaces or type aliases.

interface

Best for object shapes

  • Can be extended
  • Can be merged
  • Classic OOP style
type

Best for unions & computed types

  • Union & intersection
  • Mapped types
  • More flexible
Use interface for objects, type for everything else
interfaces.ts 
// Interface — define object shape
interface User {
  id: number;
  name: string;
  email: string;
  avatar?: string; // optional
}

// Extend an interface
interface Admin extends User {
  role: "admin";
  permissions: string[];
}

// Type alias — unions, intersections
type ID = string | number;
type Theme = "light" | "dark";

// Intersection type
type Timestamped = {
  createdAt: Date;
  updatedAt: Date;
};

type Post = {
  title: string;
  body: string;
} & Timestamped;

// Use with functions
function createUser(data: User): User {
  return { ...data };
}
Types 3

Generics

Write code that works with any type

Generics let you write flexible, reusable code without losing type safety.

Without generics fn(x: any): any
→
With generics fn<T>(x: T): T

Common uses:

  • Functions that wrap values
  • Data structures (lists, maps, sets)
  • API response wrappers
  • Constrained types with extends
generics.ts 
// Generic function
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

first([1, 2, 3]);       // number
first(["a", "b"]);     // string

// Generic interface
interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

type UserResponse = ApiResponse<User>;
type PostResponse = ApiResponse<Post[]>;

// Constrained generic
function getProperty<T, K extends keyof T>(
  obj: T,
  key: K
): T[K] {
  return obj[key];
}

const user = { name: "Alice", age: 28 };
getProperty(user, "name"); // string
getProperty(user, "foo");  // Error!
Types 4

Type Narrowing

Refine types with control flow

TypeScript narrows types inside conditional blocks — it knows more about your data after checks.

typeof x === "string"

Narrows to primitive types

x instanceof Date

Narrows to class instances

"key" in obj

Narrows to objects with that property

x !== null

Eliminates null/undefined

Advanced:

  • Discriminated unions use a shared literal field
  • Custom type guards with is keyword
narrowing.ts 
// typeof narrowing
function format(val: string | number): string {
  if (typeof val === "string") {
    return val.toUpperCase(); // string
  }
  return val.toFixed(2);      // number
}

// Discriminated union
type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rect"; w: number; h: number };

function area(s: Shape): number {
  switch (s.kind) {
    case "circle":
      return Math.PI * s.radius ** 2;
    case "rect":
      return s.w * s.h;
  }
}

// Custom type guard
function isString(x: unknown): x is string {
  return typeof x === "string";
}
Types 5

Utility Types

Built-in type transformations

TypeScript ships with powerful built-in types that transform existing types.

Partial<T>

All properties optional

Required<T>

All properties required

Pick<T, K>

Select specific properties

Omit<T, K>

Remove specific properties

Record<K, V>

Map keys to values

Readonly<T>

Prevent mutation

utility.ts 
interface User {
  id: number;
  name: string;
  email: string;
}

// Partial — for update payloads
function updateUser(
  id: number,
  data: Partial<User>
) { /* ... */ }

updateUser(1, { name: "Bob" }); // OK

// Pick — select fields
type UserPreview = Pick<User, "id" | "name">;
// { id: number; name: string }

// Omit — remove fields
type CreateUser = Omit<User, "id">;
// { name: string; email: string }

// Record — typed dictionaries
type Roles = "admin" | "user" | "guest";
const perms: Record<Roles, string[]> = {
  admin: ["read", "write", "delete"],
  user:  ["read", "write"],
  guest: ["read"],
};

// Readonly — immutable
const config: Readonly<User> = {
  id: 1, name: "Alice", email: "a@b.c"
};
// config.name = "Bob"; // Error!
