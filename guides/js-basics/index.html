Intro

Why Learn JavaScript?

JavaScript is the only language that runs natively in every browser.

It powers:

  • Interactive websites and web apps
  • Server-side code with Node.js
  • Mobile apps (React Native, Ionic)
  • Desktop apps (Electron)

Modern JS gives you:

  • Block-scoped variables with let and const
  • Arrow functions and destructuring
  • Async/await for clean asynchronous code
  • Modules for organized, reusable code
hello.js 
// JavaScript runs everywhere
const greet = (name) => {
  return `Hello, ${name}!`;
};

console.log(greet("World"));
// "Hello, World!"
modern.js 
// Modern JS features
const users = [
  { name: "Alice", age: 28 },
  { name: "Bob", age: 32 },
];

const names = users.map((u) => u.name);
// ["Alice", "Bob"]
Basics 1

Variables & Data Types

Variables store values. Use const by default, let when you need reassignment.

String

"hello" or `template`

Number

42, 3.14, NaN

Boolean

true or false

null / undefined

Intentional empty vs unset

Avoid var — it has confusing scoping rules
variables.js 
// const — cannot be reassigned
const API_URL = "https://api.example.com";
const MAX_RETRIES = 3;

// let — can be reassigned
let count = 0;
count = count + 1; // OK

// Data types
const name = "Alice";        // string
const age = 28;              // number
const active = true;         // boolean
const empty = null;          // null
let notSet;                   // undefined

// Template literals
const msg = `${name} is ${age} years old`;

// typeof checks the type
typeof name;  // "string"
typeof age;   // "number"
Basics 2

Functions

Functions are reusable blocks of code. JS has multiple ways to define them.

function declaration

Hoisted — can be called before it's defined

function expression

Assigned to a variable — not hoisted

arrow function

Short syntax, lexical this

Key concepts:

  • Parameters receive values, arguments pass them
  • Default parameters: fn(x = 10)
  • Rest parameters: fn(...args)
  • Functions are first-class — pass them like values
functions.js 
// Function declaration (hoisted)
function add(a, b) {
  return a + b;
}

// Arrow function (concise)
const multiply = (a, b) => a * b;

// Default parameters
const greet = (name = "stranger") => {
  return `Hello, ${name}!`;
};

// Rest parameters
const sum = (...numbers) => {
  return numbers.reduce((a, b) => a + b, 0);
};

sum(1, 2, 3, 4); // 10

// Functions as arguments (callbacks)
const nums = [1, 2, 3];
const doubled = nums.map(n => n * 2);
// [2, 4, 6]
Basics 3

Arrays & Objects

Arrays hold ordered lists. Objects hold key-value pairs. Together they model any data.

Array

Ordered, indexed from 0

  • .map() — transform
  • .filter() — select
  • .reduce() — accumulate
Object

Key-value pairs, unordered

  • obj.key — dot access
  • obj["key"] — bracket access
  • { ...obj } — spread/copy
arrays.js 
// Arrays
const fruits = ["apple", "banana", "cherry"];
fruits.push("date");    // add to end
fruits[0];              // "apple"
fruits.length;          // 4

// Array methods (immutable)
const prices = [10, 20, 30, 5];
const expensive = prices.filter(p => p > 15);
const total = prices.reduce((sum, p) => sum + p, 0);

// Destructuring
const [first, ...rest] = fruits;
objects.js 
// Objects
const user = {
  name: "Alice",
  age: 28,
  skills: ["JS", "CSS"],
};

// Destructuring
const { name, age } = user;

// Spread — shallow copy
const updated = { ...user, age: 29 };

// Optional chaining
const city = user.address?.city;
// undefined (no error)
Basics 4

Control Flow

Control flow decides which code runs and how many times.

if / else

Branch based on a condition

for / for...of

Repeat over a range or collection

ternary ?:

Inline conditional expression

Important:

  • Use === (strict equality), not ==
  • for...of for arrays, for...in for object keys
  • Prefer .map() / .filter() over manual loops
control.js 
// if / else if / else
const score = 85;

if (score >= 90) {
  console.log("A");
} else if (score >= 80) {
  console.log("B");
} else {
  console.log("C");
}

// Ternary operator
const label = score >= 80 ? "pass" : "fail";

// for...of — iterate over values
const colors = ["red", "green", "blue"];

for (const color of colors) {
  console.log(color);
}

// Strict equality — always use ===
0 == "";    // true  (type coercion!)
0 === "";   // false (safe)

// Nullish coalescing
const val = null ?? "default";
// "default"
Basics 5

DOM Manipulation

The DOM is JavaScript's interface to the HTML page. Read, create, and modify elements.

1
Select

Find elements with querySelector

2
Modify

Change text, attributes, or styles

3
Listen

React to clicks, input, and other events

Best practices:

  • Use querySelector over getElementById
  • Prefer textContent over innerHTML for safety
  • Use event delegation for dynamic lists
dom.js 
// Select elements
const btn = document.querySelector(".btn");
const items = document.querySelectorAll("li");

// Modify content
btn.textContent = "Click me";
btn.classList.add("btn--active");
btn.setAttribute("disabled", true);

// Listen for events
btn.addEventListener("click", (event) => {
  console.log("Clicked!", event.target);
});

// Create elements
const el = document.createElement("p");
el.textContent = "New paragraph";
document.body.appendChild(el);

// Loop over NodeList
items.forEach((item) => {
  item.style.color = "blue";
});
Basics 6

Async & Promises

JavaScript is single-threaded but non-blocking. Async code lets you wait without freezing.

Callbacks

The old way — leads to "callback hell"

Promises

Chainable with .then()

async/await

Looks synchronous, works asynchronously

Key points:

  • Always handle errors with try/catch
  • Use Promise.all() for parallel requests
  • fetch() is the modern API for HTTP requests
async.js 
// Promise with .then()
fetch("https://api.example.com/users")
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async/await — cleaner syntax
async function getUsers() {
  try {
    const res = await fetch("/api/users");
    const data = await res.json();
    return data;
  } catch (err) {
    console.error("Failed:", err);
  }
}

// Parallel requests
const [users, posts] = await Promise.all([
  fetch("/api/users").then(r => r.json()),
  fetch("/api/posts").then(r => r.json()),
]);
