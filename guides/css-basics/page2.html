Step 1

Start with HTML

You can't style what you don't understand

CSS targets HTML elements — learn the structure first.

  • Understand semantic tags: <header>, <main>, <section>
  • Know the difference between block and inline
  • Write valid, well-nested HTML before adding styles
  • Use the right element for the job (not <div> for everything)
💡

Build 2-3 pages with zero CSS first. If the HTML makes sense without styles, you've got a solid foundation.

good-structure.html 
<!-- Semantic HTML = easier CSS -->
<header>
  <nav>
    <a href="/">Logo</a>
    <ul>
      <li><a href="/about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h1>Page Title</h1>
    <p>Content goes here.</p>
  </article>
</main>

<footer>
  <p>&copy; 2026</p>
</footer>
Step 2

Learn the Box Model

Everything in CSS is a box

Every element is a rectangle with four layers.

margin
border
padding
content

Key rule:

  • Always use box-sizing: border-box
  • Padding adds space inside the border
  • Margin adds space outside the border
  • Width includes padding + border (with border-box)
box-model.css 
/* Always start with this */
*, *::before, *::after {
  box-sizing: border-box;
}

/* Now widths are predictable */
.card {
  width: 300px;
  padding: 20px;
  border: 2px solid gray;
  /* Total width = 300px (not 344px!) */
}

/* Without border-box: */
/* width = 300 + 20*2 + 2*2 = 344px */

/* With border-box: */
/* width = 300px (padding + border inside) */
Step 3

Master Selectors & Specificity

Know how CSS decides which rule wins

Selectors target elements. Specificity decides conflicts.

0-0-1 p Element
0-1-0 .card Class
1-0-0 #header ID
∞ style="" Inline

Rules to remember:

  • Prefer classes over IDs for styling
  • Keep selectors short and flat
  • Avoid !important — it's a last resort
specificity.css 
/* 0-0-1 — element selector */
p {
  color: black;
}

/* 0-1-0 — class selector (wins!) */
.intro {
  color: blue;
}

/* 0-2-0 — two classes (even higher) */
.card .intro {
  color: purple;
}

/* 1-0-0 — ID (avoid for styling) */
#header {
  color: red;
}

/* Same specificity? Last one wins */
.text { color: green; }
.text { color: orange; } /* wins */
Step 4

Understand the Cascade

CSS stands for Cascading Style Sheets

The cascade is how the browser resolves competing rules.

1
Origin & Importance

User agent → Author → Author !important

2
Specificity

ID > Class > Element

3
Source Order

Later rules override earlier ones

💡

If a style isn't applying, check these three in order — the answer is almost always here.

cascade.css 
/* Source order matters */
.btn {
  background: gray;
}

.btn {
  background: blue; /* wins — later */
}

/* Specificity beats source order */
.nav .btn {
  background: green; /* 0-2-0 wins */
}

.btn {
  background: red; /* 0-1-0 loses */
}

/* Inheritance — some props cascade down */
body {
  font-family: sans-serif; /* inherited */
  color: #333;        /* inherited */
  border: 1px solid;  /* NOT inherited */
}
Step 5

Learn Flexbox & Grid

The two modern layout systems

Forget floats. These two handle all layout needs.

Flexbox

One-dimensional — rows or columns

  • Navbars, toolbars
  • Centering content
  • Distributing space
Grid

Two-dimensional — rows and columns

  • Page layouts
  • Card grids
  • Complex alignment
flex.css 
/* Flexbox — one direction */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 1rem;
}

/* Center anything */
.center {
  display: flex;
  place-items: center;
}
grid.css 
/* Grid — two dimensions */
.page {
  display: grid;
  grid-template-columns: 250px 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

/* Responsive card grid */
.cards {
  display: grid;
  grid-template-columns:
    repeat(auto-fill, minmax(280px, 1fr));
  gap: 1.5rem;
}
Step 6

Use Browser DevTools

Your most powerful learning tool

Inspect, edit, and debug CSS in real time.

F12 Open DevTools
Ctrl+Shift+C Inspect element
Styles panel See applied CSS rules
Computed tab See final values + box model

What to look for:

  • Crossed-out rules = overridden by specificity
  • Grayed-out values = inherited
  • Toggle properties on/off to experiment
  • Edit values live in the browser
DevTools — Styles Panel 
/* What you see in DevTools: */

/* Active rule */
.card {
  padding: 1.5rem;      /* ✓ applied */
  background: white;    /* ✓ applied */
  color: black;        /* ✗ overridden */
}

/* Winning rule (higher specificity) */
.theme-dark .card {
  color: white;         /* ✓ wins */
}

/* Inherited from parent */
body {
  font-family: sans-serif; /* inherited ↓ */
}
Step 7

Practice with Real Projects

Reading docs is not enough — build things

The fastest way to learn CSS is to build and break things.

Beginner

Recreate a simple landing page from a screenshot

Intermediate

Build a responsive card grid with Flexbox or Grid

Advanced

Clone a real website's layout (header, sidebar, content)

Challenge

Build a dark/light theme toggle with CSS custom properties

💡

Don't aim for perfection — aim for understanding. Every broken layout teaches you something.
