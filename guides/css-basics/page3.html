Intro

Why Learn Modern CSS Syntax?

CSS has evolved dramatically since 2020.

Before:

  • Needed Sass/Less for variables & nesting
  • No way to query a parent element
  • Color manipulation required JS or preprocessors

Now:

  • Native variables, nesting, and scoping
  • Parent selectors with :has()
  • Built-in color functions and container queries
before.scss 
/* Old way — needed Sass */
$primary: #6366f1;

.card {
  background: $primary;

  .title {
    font-size: 1.5rem;
  }
}
now.css 
/* Now — plain CSS! */
.card {
  background: var(--color-primary);

  .title {
    font-size: 1.5rem;
  }
}
Syntax 1

Custom Properties (Variables)

Define once, use everywhere — and they cascade.

--color-primary : #6366f1
Custom property Value

Key points:

  • Declared with -- prefix
  • Accessed via var(--name)
  • Cascade & inherit like any CSS property
  • Can be overridden per selector or media query
Unlike Sass variables, they are live & reactive
tokens.css 
/* Define tokens on :root */
:root {
  --color-primary: #6366f1;
  --color-surface: #ffffff;
  --spacing-md: 1rem;
  --radius: 0.5rem;
}

/* Override for dark mode */
[data-theme="dark"] {
  --color-surface: #0f172a;
}

/* Use them anywhere */
.card {
  background: var(--color-surface);
  padding: var(--spacing-md);
  border-radius: var(--radius);
}

/* Fallback value */
.alert {
  color: var(--color-danger, #ef4444);
}
Syntax 2

Native CSS Nesting

Write nested rules directly in CSS — no preprocessor needed.

Flat CSS .card { }
.card .title { }
→
Nested CSS .card {
  .title { }
}

Rules:

  • Nested selectors start a new scope
  • Use & to reference the parent selector
  • Media queries can be nested too
  • Supported in all modern browsers (2023+)
card.css 
.card {
  padding: 1.5rem;
  background: var(--color-surface);

  /* Nested element */
  .title {
    font-size: 1.25rem;
    font-weight: 700;
  }

  /* & references the parent */
  &:hover {
    box-shadow: 0 4px 12px rgb(0 0 0 / 0.1);
  }

  /* Modifier */
  &.card--featured {
    border: 2px solid var(--color-primary);
  }

  /* Nested media query */
  @media (width < 768px) {
    padding: 1rem;
  }
}
Syntax 3

Modern Selectors — :is() :where() :has()

Three pseudo-classes that simplify complex selectors.

:is()

Matches any selector in the list. Takes the highest specificity of its arguments.

:where()

Same as :is() but with zero specificity. Great for resets.

:has()

The "parent selector" — style a parent based on its children.

selectors.css 
/* :is() — shorter grouped selectors */
:is(h1, h2, h3) {
  line-height: 1.2;
  font-weight: 700;
}

/* :where() — zero specificity */
:where(.reset) a {
  color: inherit;
  text-decoration: none;
}

/* :has() — the parent selector! */
.card:has(img) {
  grid-template-rows: auto 1fr;
}

/* Style form when input is invalid */
form:has(:invalid) {
  border-color: var(--color-danger);
}

/* Next sibling of checked input */
input:checked + label {
  font-weight: bold;
}
Syntax 4

Container Queries

Respond to a component's size, not the viewport.

Media query

Responds to viewport width

vs
Container query

Responds to parent width

How it works:

  • Define a containment context with container-type
  • Query it with @container
  • Name containers for targeted queries
  • Components adapt to where they're placed
container.css 
/* 1. Define the container */
.sidebar {
  container-type: inline-size;
  container-name: sidebar;
}

/* 2. Query the container */
@container sidebar (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 120px 1fr;
    gap: 1rem;
  }
}

@container sidebar (max-width: 399px) {
  .card {
    display: flex;
    flex-direction: column;
  }
}
Syntax 5

Modern Color Functions

Perceptually uniform color spaces and built-in mixing.

oklch()

Lightness, Chroma, Hue — perceptually uniform

color-mix()

Blend two colors in any color space

Relative color

Derive colors from existing values

colors.css 
/* oklch — perceptually uniform */
:root {
  --brand: oklch(65% 0.25 265);
}

/* color-mix — blend in any space */
.btn:hover {
  background: color-mix(
    in oklch,
    var(--brand) 80%,
    black
  );
}

/* Semi-transparent from any color */
.overlay {
  background: color-mix(
    in srgb,
    var(--brand) 40%,
    transparent
  );
}

/* Relative color syntax */
.light-variant {
  color: oklch(from var(--brand) 85% c h);
}
Syntax 6

Logical Properties

Write direction-aware CSS that works in any language.

Physical → Logical
margin-left → margin-inline-start
margin-right → margin-inline-end
padding-top → padding-block-start
width → inline-size
height → block-size

Why it matters:

  • RTL support without extra code
  • Consistent with writing direction
  • Modern approach to internationalization
logical.css 
/* Physical — breaks in RTL */
.card {
  margin-left: 1rem;
  padding-top: 2rem;
  border-right: 2px solid blue;
}

/* Logical — works in any direction */
.card {
  margin-inline-start: 1rem;
  padding-block-start: 2rem;
  border-inline-end: 2px solid blue;
}

/* Shorthand — inline = left/right */
.container {
  margin-inline: auto;
  padding-inline: 1.5rem;
  max-inline-size: 1200px;
}
