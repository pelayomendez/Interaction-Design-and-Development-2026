Intro

Why Learn the Terminal?

The terminal gives you direct, text-based control over your computer — no clicking, no menus.

You can:

  • Navigate your file system at lightning speed
  • Automate repetitive tasks with scripts
  • Use developer tools like Git, npm, and Docker
  • Connect to remote servers via SSH

Terminology:

  • Terminal — the window/app you type in
  • Shell — the program interpreting your commands (bash, zsh)
  • CLI — Command Line Interface
  • Prompt — the text waiting for your input (e.g. $)
terminal 
# Your first commands
$ whoami
alice

$ date
Wed Apr  9 10:30:00 CEST 2026

$ echo "Hello, Terminal!"
Hello, Terminal!

$ pwd
/Users/alice
anatomy 
# Anatomy of a command
$ command -flags arguments

# Example:
$ ls -la /Users/alice
#  ^    ^     ^
#  cmd  flags  argument
Basics 1

Navigation

Move around your file system with three essential commands.

pwd

Print Working Directory — shows where you are

ls

List files and folders in the current directory

cd

Change Directory — move to another folder

Path shortcuts:

  • ~ — your home directory
  • . — current directory
  • .. — parent directory
  • / — root of the file system
Press Tab to autocomplete paths and commands
navigation.sh 
# Where am I?
$ pwd
/Users/alice

# List files (simple)
$ ls
Desktop  Documents  Downloads

# List with details + hidden files
$ ls -la
drwxr-xr-x  5 alice staff  160 Apr 9 10:00 .
drwxr-xr-x  3 alice staff   96 Apr 9 09:00 ..
-rw-r--r--  1 alice staff  220 Apr 9 10:00 .bashrc
drwxr-xr-x  4 alice staff  128 Apr 9 10:00 projects

# Move around
$ cd projects         # go into folder
$ cd ..               # go back up
$ cd ~                # go home
$ cd ~/Desktop/work   # absolute path
$ cd -                # go to previous dir
Basics 2

File Operations

Create, copy, move, and delete files and folders from the command line.

touch / mkdir

Create files and directories

cp / mv

Copy and move (or rename) files

rm / rmdir

Remove files and empty directories

Be careful:

  • rm deletes permanently — no trash bin!
  • rm -rf is dangerous — deletes recursively without asking
  • Always double-check paths before deleting
files.sh 
# Create files and directories
$ touch index.html         # create a file
$ mkdir my-project         # create a folder
$ mkdir -p src/components  # create nested dirs

# Copy
$ cp file.txt copy.txt    # copy a file
$ cp -r src/ backup/      # copy folder recursively

# Move / Rename
$ mv old.txt new.txt     # rename a file
$ mv file.txt src/       # move to folder

# Delete
$ rm file.txt             # delete a file
$ rmdir empty-folder      # delete empty dir
$ rm -r folder/           # delete dir + contents
Basics 3

Reading Files

View file contents without opening an editor — fast and flexible.

Quick View

Print the whole file or parts of it

  • cat — print all
  • head — first N lines
  • tail — last N lines
Interactive

Scroll through large files

  • less — scroll up/down
  • /text — search inside
  • q — quit
reading.sh 
# Print entire file
$ cat readme.md

# First 10 lines (default)
$ head readme.md

# First 5 lines
$ head -n 5 readme.md

# Last 20 lines
$ tail -n 20 server.log

# Follow a log file in real time
$ tail -f server.log

# Scroll through a large file
$ less big-file.txt
# Use arrows to scroll, /word to search, q to quit

# Count lines, words, characters
$ wc -l readme.md
42 readme.md
Basics 4

Searching

Find files by name or search inside files for text patterns.

find

Search by file name, type, or size

grep

Search inside files for text patterns

which

Find where a command is installed

Useful flags:

  • grep -r — recursive (search in subdirs)
  • grep -i — case insensitive
  • grep -n — show line numbers
  • find -name "*.js" — wildcards
search.sh 
# Find files by name
$ find . -name "*.html"
./index.html
./about/index.html

# Find only directories
$ find . -type d -name "src"

# Search text inside files
$ grep "TODO" app.js
12:  // TODO: add error handling

# Recursive + case insensitive + line numbers
$ grep -rin "error" src/
src/api.js:8:  throw new Error("Not found");
src/utils.js:14:  console.error(msg);

# Where is a command?
$ which node
/usr/local/bin/node
Basics 5

Permissions & Processes

Control who can access files and manage running programs.

r
Read

View file contents or list a directory

w
Write

Modify file contents or add/remove files in a dir

x
Execute

Run a file as a program or enter a directory

Key commands:

  • chmod — change permissions
  • sudo — run as superuser (admin)
  • ps / top — view running processes
  • kill — stop a process by its PID
permissions.sh 
# Understanding permissions: rwxr-xr--
#   rwx   r-x   r--
#   user  group  others

# Make a script executable
$ chmod +x deploy.sh

# Set specific permissions (owner: rwx, others: r)
$ chmod 744 deploy.sh

# Run as admin
$ sudo apt install nodejs

# List running processes
$ ps aux | grep node
alice 1234 0.5  1.2 node server.js

# Stop a process
$ kill 1234

# Force stop
$ kill -9 1234
Basics 6

Pipes & Redirects

Chain commands together and control where output goes — the real power of the terminal.

| (pipe)

Send one command's output into the next command

> (redirect)

Write output to a file (overwrites)

>> (append)

Append output to a file

Pro tips:

  • Ctrl + C — cancel a running command
  • Ctrl + L — clear the screen
  • Up/Down arrows — cycle through history
  • history — list past commands
pipes.sh 
# Pipe: chain commands with |
$ cat names.txt | sort
# Sorts the lines alphabetically

$ cat names.txt | sort | uniq
# Sort + remove duplicates

$ ls -la | grep ".js"
# List only JS files

$ ps aux | grep node | wc -l
# Count node processes

# Redirect output to a file
$ echo "hello" > file.txt    # overwrite
$ echo "world" >> file.txt   # append

# Save command output
$ ls -la > filelist.txt

# Combine: find TODOs and save to file
$ grep -rn "TODO" src/ > todos.txt
