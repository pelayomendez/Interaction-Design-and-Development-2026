Intro

Why CSS Organization Matters

CSS scales poorly without structure.

It leads to:

  • Style conflicts across components
  • Hard-to-maintain overrides
  • Inconsistent UI across teams

Goal:

  • Predictable styles
  • Isolated scope
  • Reusable across the codebase
chaos.css 
/* ⚠️ Without organization */
.container {
  width: 100%;
}

/* 200 lines later… */
.container {
  width: 90%; /* 😬 override */
}

/* Another file… */
.container {
  width: 80% !important; /* 🔥 */
}
Technique 1

Component-Based CSS

Co-locate styles with components.

📁 Button/
📄 Button.tsx
📄 Button.module.css
📁 Card/
📄 Card.tsx
📄 Card.module.css

Benefits:

  • Isolation by default
  • Easier refactoring
  • Clear ownership per component
👉 Works great with React + MFE
Button.module.css 
/* Styles live next to the component */
.button {
  display: inline-flex;
  align-items: center;
  padding: 0.5rem 1rem;
  border-radius: 0.375rem;
  font-weight: 600;
  cursor: pointer;
}

.button--primary {
  background: var(--color-primary);
  color: #fff;
}
Button.tsx 
import styles from './Button.module.css';

export const Button = ({ children }) => (
  <button className={styles.button}>
    {children}
  </button>
);
Technique 2

CSS Modules

Scoped class names, automatically.

You write .button
→
Browser gets .button__3xQp7

When to use:

  • Medium to large apps
  • Teams with multiple contributors
  • When you need zero class-name collisions
layout.module.css 
/* Both files can use .container */
/* — no collisions! */

.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 1.5rem;
}

/* sidebar.module.css */
.container {
  width: 280px;
  flex-shrink: 0;
}
/* ✅ Both compile to unique hashes */
Technique 3

Utility-First CSS (Tailwind)

Compose UI with small utility classes.

Pros ✅

  • Fast development
  • Consistent spacing & scale
  • No naming bikeshedding

Cons ⚠️

  • Can get messy without discipline
  • Harder to read for newcomers
  • HTML can get verbose
Card.tsx 
// Traditional CSS approach
<div className="card">
  <h2 className="card__title">Hello</h2>
</div>

// Tailwind approach
<div className="
  flex flex-col gap-4
  p-6 rounded-xl
  bg-white shadow-md
  border border-gray-100
">
  <h2 className="text-lg font-semibold
           text-gray-900">
    Hello
  </h2>
</div>
Technique 4

Design System / Tokens

Centralize your visual language.

🎨 Colors --color-primary --color-surface
📐 Spacing --spacing-sm --spacing-md
✍️ Typography --font-size-base --font-weight-bold

Benefits:

  • Consistency across the app
  • Easy theming (dark mode!)
  • Aligns with Figma / design team
tokens.css 
:root {
  /* Colors */
  --color-primary: #6366f1;
  --color-primary-hover: #4f46e5;
  --color-surface: #ffffff;
  --color-text: #111827;

  /* Spacing */
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 2rem;

  /* Typography */
  --font-size-base: 1rem;
  --font-size-lg: 1.25rem;
  --font-weight-bold: 700;
}

[data-theme="dark"] {
  --color-surface: #0f172a;
  --color-text: #f1f5f9;
}
Technique 5

BEM — for Global CSS

A naming convention that makes global CSS predictable.

card __ title -- highlighted
Block Element Modifier

Why use it:

  • Makes global CSS predictable
  • Reduces specificity issues
  • Self-documenting selectors
👉 Useful if not using CSS Modules
card.css 
/* Block */
.card {
  padding: 1.5rem;
  border-radius: 0.5rem;
  background: white;
}

/* Element */
.card__title {
  font-size: 1.25rem;
  font-weight: 700;
}

/* Modifier */
.card__title--highlighted {
  color: var(--color-primary);
  text-decoration: underline;
}
Technique 6

Layering — Modern CSS

Use @layer to control the cascade explicitly.

↑ highest priority utilities
components
↓ lowest priority base

Goal:

  • Avoid specificity wars
  • Make overrides intentional
  • Works well with third-party CSS
main.css 
/* Declare layer order first */
@layer base, components, utilities;

@layer base {
  *, *::before, *::after {
    box-sizing: border-box;
  }
  body {
    margin: 0;
    font-family: sans-serif;
  }
}

@layer components {
  .button {
    padding: 0.5rem 1rem;
  }
}

@layer utilities {
  .mt-4 {
    margin-top: 1rem;
  }
}
